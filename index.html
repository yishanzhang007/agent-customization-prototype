Configuring intent-specific instructions

Four different UX directions for letting clinic staff author what the agent should do for each kind of call. Same underlying data model, fundamentally different mental models.

Option 3 · v3 — conditional editor
→

Hand-off vs. handle-it

Right panel collapses to match the action. Pick a hand-off action (Transfer / Direct / SMS / End) and you only fill in a destination plus an optional one-liner. Pick a handle-it action (Take a message / Answer with KB / Schedule / Prep for approval) and a prompt textarea appears. Most intents become a 2-field config.

conditional fields prompt only when AI handles 8 action types · grouped click to switch instantly

How to use these

Each prototype uses the same sample data (5 default intents + 1 custom: "Clinical trial inquiry"). They're not connected to a backend; nothing persists. The goal is to compare mental models — which one feels like the right authoring surface for a clinic admin?
